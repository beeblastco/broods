# GitHub

Use this skill for GitHub work.

Before acting externally, confirm the target owner/repo and the intended change. Prefer read-only inspection first:

- Search or open the requested repository, issue, pull request, workflow run, or file.
- Summarize facts with links or exact identifiers.
- For write actions, draft the intended change first and ask for confirmation unless the user explicitly asked you to perform it.

When code execution is useful, use the sandbox workspace for scratch files and keep durable notes in `MEMORY.md` or `TASKS.md` only when the user asks you to remember them.
